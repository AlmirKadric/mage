Class Archivist

Abstracted data store access interface

In general, you will be accessing an archivist instance through a state object:

state.archivist.set('player', { userId: userId }, { name: 'someone' });

list

  • list(topicName: string, partialIndex: IArchivistIndex, options: IArchivistListOptions | undefined, callback: ArchivistListCallback): void
  • list(topicName: string, partialIndex: IArchivistIndex, callback: ArchivistListCallback): void
    • Defined in mage.d.ts:269

    Scan the backend vault for matching indexes, and return them

    In this case, the index can be partial; for instance, { userId: 1 }, would match all the following indexes:

    { userId: 1, somethingElse: 'hi'}
{ userId: 1, somethingElse: 'hello'}

    Note that this API returns the list of matching indexes, not the data they hold; to fetch the data, you will want to call mget using the returned list of indexes.

    See https://mage.github.io/mage/#key-based-filtering for more details.
